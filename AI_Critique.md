# AI Critique (200-300 Words)
In this Domain Testing and Boundary Value Analysis homework (HW02), I used AI tools, especially AI agent to generate test cases, boundary value analysis, and domain testing. When I work with the AI, I can see that the AI exhibited several benefits to my homework completion, such as generating test cases rapidly without the human manually carrying out the work, plus AI can also spot code defects quickly rather than reading the whole codebase manually. However, the AI also showed several weaknesses when testing. First, the AI tend to over-focus on the specification where it provided test cases that match the intended specification on README.md, while forgetting that the frontend implementation is completely missing them, even if the AI noticed the defects, which are found when I do FR-01 and FR-07. Second, the AI may also misses test cases due to context memory limitation, as evidenced when I perform Domain Testing for FR-13. And finally, Due to the inherent probabilistic nature of LLMs, there are several situations where the AI produces an outcome with "or" that violates software testing principle. What I learned is AI is only good at producing mass raw outputs, while the actual outputs need human validation.
